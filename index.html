Laravel Role-Based Auth

Laravel App with Role-Based Authentication

This guide explains how to create a Laravel 12 application with role-based authentication using best practices.

Step 1: Create Laravel Project
composer create-project laravel/laravel laravel-auth-app
Step 2: Configure Database

Edit the .env file:


DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_auth
DB_USERNAME=root
DB_PASSWORD=
Step 3: Install Laravel Breeze (Authentication)

composer require laravel/breeze --dev
php artisan breeze:install
npm install && npm run dev
php artisan migrate
Step 4: Create Roles Table

php artisan make:model Role -m

Edit the migration file:


Schema::create('roles', function (Blueprint $table) {
    $table->id();
    $table->string('name'); // e.g., admin, user
    $table->timestamps();
});

Run migration:

php artisan migrate
Step 5: Create Migration to Add Column in Users Table

Generate a new migration:

php artisan make:migration add_role_id_to_users_table --table=users

Edit the migration file:


Schema::table('users', function (Blueprint $table) {
    $table->unsignedBigInteger('role_id')->default(2)->after('id');
    // Optional: add foreign key constraint
    // $table->foreign('role_id')->references('id')->on('roles');
});

Run the migration:

php artisan migrate
Step 6: Middleware for Role

php artisan make:middleware RoleMiddleware

In handle() method:


if (auth()->user()->role->name !== $role) {
    abort(403);
}
Step 7: Protect Routes

Route::middleware(['auth', 'role:admin'])->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index']);
});
✅ You now have a Laravel app with role-based authentication!
